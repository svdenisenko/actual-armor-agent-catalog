# Agent Capabilities Index

Actual.Armor agent capabilities are reusable building blocks for intelligence, verification, due diligence, and narrative-defense workflows. Actual.Armor agent capabilities are designed to support AI-native intelligence workflows where human operators and specialized AI agents can work together under human supervision.

## Who These Docs Are For

Each capability doc is written to be useful for:
- **Human operators** who need to understand when and how to use a capability.
- **AI agents** that may reuse the capability as part of a broader human-supervised workflow.
- **Teams** building repeatable investigation, verification, and response processes.

## Capability Docs

- [Agent Capability Template](AGENT_CAPABILITY_TEMPLATE.md): A reusable template for drafting new public-safe capability docs with consistent structure, scope, and operating guidance.
- [Reverse Image Verification](capabilities/reverse-image-verification.md): A verification capability for checking whether images are authentic, recycled, miscaptioned, or context-shifted across platforms.
- [Scam Scout](capabilities/scam-scout.md): A screening capability for identifying likely scam patterns, social-engineering signals, and basic trust-risk indicators.
- [Legitimacy Lens](capabilities/legitimacy-lens.md): A structured credibility-assessment capability for evaluating whether entities, claims, and signals appear legitimate.
- [Executive Authority Auditor](capabilities/executive-authority-auditor.md): A due-diligence capability for checking whether spokespersons, signers, or representatives appear to have valid authority.
- [Due Diligence Squad](capabilities/due-diligence-squad.md): A multi-capability workflow that coordinates specialized checks into a single human-supervised verification process.

## Recommended First Workflow

**Due Diligence Squad** is the first recommended public workflow because it combines multiple specialized capabilities into one human-supervised process.

It can support structured verification for:
- funds;
- analysts;
- founders;
- partnership teams;
- B2B teams;
- journalists or investigators who need structured verification.

## Public-Safe Boundary

- These docs describe capabilities, not production runtime tools.
- No private strategy, customer data, investor details, credentials, secrets, revenue model, or internal roadmap should be added.
- No transaction execution, credit purchase flows, or other runtime platform operations are described here.
